## 1. What shipped (plain language)

This was the final Building phase: it polished the editor we already had rather than adding anything new. A client can now do everything with the keyboard alone — tab through every control with a clear focus outline, jump past the menus with a "skip to main content" link, switch language tabs with the arrow keys, and confirm a delete in a proper pop-up dialog that traps focus and closes on Escape. Every button and field has a clear name a screen reader can read; while the portal is saving, publishing, deleting, or uploading it says so out loud and disables the other buttons; and the "nothing here yet / something needs attention" screens all look and read the same. We checked every text colour against the dark background and nudged two greys lighter so they meet the accessibility contrast standard, and we honour the "reduce motion" setting. None of this changed how content is saved or who can touch which client's site. To prove it stays accessible, there are now automated accessibility tests that fail the build if a violation creeps in.

## 3. Decisions I made during this phase

Logged in full in `dashboard-Decisions.md` (the 2026-06-28 "B.08 accessibility & polish choices" entry). Summary:

- **axe tooling = `jest-axe` + `@testing-library/react`/`-dom`/`-user-event` + `jsdom`, all dev-only, pinned exact.** `jest-axe@10.0.0` is the most stable, runner-agnostic axe matcher and works under Vitest 4 with a one-file type augmentation (`test/setup/vitest-axe.d.ts`); chosen over the far-less-established `vitest-axe@0.1.0`. · Alternative rejected: `vitest-axe`.
- **Two Vitest projects (unit/Node + a11y/jsdom) sharing one alias set**, replacing the single Node config — so the crypto/isolation tests stay in Node while component tests get a DOM. · Chosen over the Vitest-4-removed `environmentMatchGlobs`.
- **axe runs scoped to the WCAG 2.1 A/AA tags**; colour-contrast verified separately with a standalone calculator (jsdom doesn't apply the stylesheet, so axe can't be the contrast oracle).
- **Contrast fixes are lightness-only, within the existing grayscale/same hue** (`--muted-foreground` 45→58%, `--destructive` 60→66%, dropped `/70` alphas) — the minimum change to meet the AA the DoD mandates, honoring "adjust token application, not the palette" (no new hue).
- **Delete-confirm rebuilt as a modal Alert Dialog** (`@base-ui/react`, already a dependency) for the focus-trap/Escape/return-focus the DoD requires; bonus: modal inert-ness satisfies "disable conflicting controls during delete".
- **Locale tabs hand-rolled (not the base-ui Tabs primitive)** so every locale's panel stays mounted for a multi-locale submit, while still implementing the full tablist contract (roving tabindex + arrow keys + `aria-controls`).
- **Added a route-level loading state** (`LoadingState` + three `loading.tsx`) — the DoD lists "loading" but the portal had none.
- **Extracted a shared `MessageCard`** with a configurable heading level so the friendly states render consistently and each page keeps exactly one `h1`.

None reverses a locked decision.
